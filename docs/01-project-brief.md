# 프로젝트 기획서 (Project Brief)

## 1. 프로젝트 개요

- **프로젝트명:** HR Core Lite
- **한 줄 설명:**  
  사내 직원이 출근/퇴근을 기록하고, 휴가를 신청하고, 팀장 또는 HR 관리자가 이를 승인·관리할 수 있는 웹 기반 HR 시스템
- **개발 인원:** 학생 풀스택 3명
- **개발 기간:** 1~2주
- **전제 조건:** 디자인 시안 제공, React + Spring Boot + AWS 배포까지 포함

---

## 2. 왜 이 주제가 좋은가

이 프로젝트는 학생들이 아래 내용을 짧은 기간 안에 모두 경험할 수 있습니다.

| 학습 포인트 | 설명 |
|---|---|
| 인증/인가 | 로그인, 역할별 메뉴/권한 제어 |
| CRUD | 직원/부서/공휴일 데이터 관리 |
| 상태 전이 | 휴가 신청 → 승인/반려/취소 |
| 날짜 로직 | 근태, 휴가 기간, 월간 조회 |
| 실무 도메인 경험 | HR 시스템의 핵심 개념 체험 |
| 풀스택 협업 | 프론트/백엔드/배포를 모두 경험 |
| 배포 경험 | S3, CloudFront, EC2, RDS 구성 |

## 3. 문제 정의

사내에서는 아래와 같은 업무가 반복됩니다.

- 직원 정보 등록/수정
- 부서별 직원 관리
- 출근/퇴근 기록
- 휴가 신청 및 승인
- 팀장이 오늘 누가 쉬는지 확인
- HR 담당자가 공휴일/직원 상태 관리

이 프로젝트는 위의 업무를 **간단하지만 끝까지 동작하는 시스템**으로 구현하는 것이 목표입니다.

## 4. 목표 사용자

### 4.1 Employee (일반 직원)
- 로그인
- 내 정보 조회
- 출근/퇴근 처리
- 월간 근태 조회
- 휴가 신청/취소
- 내 휴가 신청 이력 조회

### 4.2 Manager (팀장/매니저)
- 팀원 목록 조회
- 팀원 근태 조회
- 휴가 승인/반려
- 팀 현황 대시보드 확인

### 4.3 Admin / HR (인사 관리자)
- 직원 등록/수정
- 부서 관리
- 공휴일 관리
- 전사 데이터 조회
- 필요한 경우 승인 대체 처리

## 5. 프로젝트 핵심 시나리오

### 시나리오 A. 직원 출근 처리
1. 직원이 로그인한다.
2. 대시보드에서 “출근” 버튼을 누른다.
3. 오늘 날짜 기준 근태 레코드가 생성된다.
4. 상태가 `PRESENT` 또는 `LATE`로 계산된다.
5. 화면에 출근 시각이 보인다.

### 시나리오 B. 휴가 신청과 승인
1. 직원이 휴가 신청서를 작성한다.
2. 시스템은 휴가 기간 중복 여부와 잔여 연차를 확인한다.
3. 신청이 성공하면 상태가 `REQUESTED`가 된다.
4. 매니저가 승인 목록에서 신청 건을 확인한다.
5. 매니저가 승인하면 상태가 `APPROVED`로 바뀐다.
6. 대시보드와 팀 일정에 반영된다.

### 시나리오 C. HR 관리자 데이터 관리
1. HR 관리자가 새 직원을 등록한다.
2. 부서와 매니저를 지정한다.
3. 해당 직원이 시스템에 로그인할 수 있게 된다.
4. 공휴일을 등록하면 휴가 계산/근태 화면에 반영할 수 있다.

## 6. 범위 정의

## 6.1 이번 프로젝트에서 반드시 구현할 범위 (In Scope)

- 로그인 / 내 정보 확인
- 역할 기반 화면 제어
- 직원 관리
- 부서 관리
- 출근 / 퇴근
- 월간 근태 조회
- 휴가 신청 / 승인 / 반려 / 취소
- 개인/팀 대시보드
- 공휴일 관리
- AWS 배포

## 6.2 이번 프로젝트에서 제외할 범위 (Out of Scope)

- 급여 계산
- 원천징수/4대보험 계산
- 다단계 전자결재
- 성과 평가/보상 시스템
- 채용 시스템(ATS)
- 사내 메신저/메일 자동 연동
- SSO/OAuth 연동

> 범위를 넓히면 짧은 기간 안에 완성도가 떨어질 가능성이 높습니다.  
> 이번 프로젝트는 “HR 시스템의 핵심 흐름”에 집중합니다.

## 7. 핵심 메뉴 구조

```text
/login

/dashboard
  ├─ 내 요약 카드
  ├─ 오늘 출근/퇴근 상태
  ├─ 남은 연차
  └─ 매니저인 경우 승인 대기 건수

/employees
/employees/:id
/departments
/attendance/me
/attendance/team
/leave/request
/leave/my
/approvals
/holidays
```

## 8. 화면별 목적

| 화면 | 목적 | 주요 사용자 |
|---|---|---|
| 로그인 | 인증 | 전체 |
| 대시보드 | 개인/팀 요약 정보 | 전체 |

| 부서 관리 | 부서 생성/수정 | 관리자 |
| 내 근태 조회 | 월간 출퇴근 기록 확인 | 직원 |
| 팀 근태 조회 | 팀원 근태 확인 | 매니저 |
| 휴가 신청 | 휴가 생성 | 직원 |
| 내 휴가 목록 | 신청 내역 조회/취소 | 직원 |
| 승인 대기 목록 | 승인/반려 처리 | 매니저, 관리자 |
| 공휴일 관리 | 공휴일 CRUD | 관리자 |

## 9. 성공 기준

아래 항목이 만족되면 프로젝트 성공으로 판단합니다.

- 핵심 시나리오 A/B/C가 실제로 동작한다.
- 역할에 맞지 않는 메뉴/API가 차단된다.
- 휴가 신청과 승인 흐름이 상태 변경까지 끝난다.
- 출근/퇴근 로직이 정상 동작한다.
- 프론트와 백엔드가 클라우드 환경에서 실제 연결된다.
- 배포 URL에서 발표 데모가 가능하다.
